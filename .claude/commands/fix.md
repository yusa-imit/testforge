# /fix — 버그 수정 워크플로우

버그를 분석하고 수정합니다. $ARGUMENTS에 버그 설명 또는 에러 메시지를 입력합니다.

## 실행 절차

### 1. 버그 분석

1. 에러 메시지 또는 증상 확인
2. 관련 코드 탐색
3. 재현 조건 파악
4. `.claude/memory/debugging.md` 확인 (유사 이슈 여부)

### 2. 근본 원인 파악

- 스택 트레이스 분석
- 관련 파일 읽기
- 데이터 흐름 추적
- 필요시 `debugger` 에이전트 활용

### 3. 수정 구현

1. 최소한의 변경으로 수정
2. 부작용 확인
3. 엣지 케이스 고려

### 4. 검증

```bash
bun run typecheck   # 타입 에러 없음
bun test            # 기존 테스트 통과
bun run build       # 빌드 성공
```

### 5. 디버깅 메모리 업데이트

수정 내용을 `.claude/memory/debugging.md`에 기록:

```markdown
### [날짜] [버그 제목]
- **증상**: [증상 설명]
- **원인**: [근본 원인]
- **수정**: [수정 내용]
- **파일**: [수정된 파일]
- **교훈**: [향후 참고사항]
```

### 6. 완료 보고

```
## Bug Fix Complete

### Bug: [버그 설명]
### Root Cause: [원인]
### Fix: [수정 내용]
### Files Changed: [파일 목록]
### Tests: [테스트 결과]
```

### 7. 반복 실패 처리

동일 에러가 3회 시도 후에도 지속되면:
1. `.claude/memory/debugging.md`에 상세 기록
2. 사용자에게 알리고 다른 접근법 제안
